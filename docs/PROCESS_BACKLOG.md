## Drive-sync red-team (2026-07-21)

Full threat model + all 32 confirmed findings: [`DRIVE_SYNC_REDTEAM_2026-07.md`](DRIVE_SYNC_REDTEAM_2026-07.md). Verdict: whole-Drive destruction is **strongly** defended (the `drive.file` OAuth scope + trash-only + ancestry filters); silent **partial** loss and API-abuse rate-limiting are the soft spots. #25 and #23 are already fixed; the highest-leverage remainder:

| Priority | Root cause class    | Proposal                                                                                                                                                                                                                                                                                                                                   | Status                                                                    |
| -------- | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------- |
| P1       | `missing invariant` | **Compile-time merge-policy exhaustiveness** per synced entity: `satisfies Record<keyof EncoreSong, MergePolicy>` (+ `EncorePerformance`, `LyreflyProject`) so a new synced field cannot compile without a conscious merge disposition; recovery-side union shares the map. Kills the #1/#2/#3 "new feature silently drops a field" class. | Open — highest leverage; the most likely future-feature data-loss bug     |
| P1       | `missing invariant` | **Move the auto-push gate inside `flushDriveWrite`** (fail-closed unless an explicit `intentionalReplace` token from the one legit replace path) and switch the If-Match precondition to the _last-pulled_ etag. Kills #9/#10/#11/#18 (manual "Back up" / Gesture Refresh clobber cloud on a never-pulled device — **live today**).        | Open — live partial-loss paths                                            |
| P1       | `test gap`          | **One shared Retry-After-honoring, bounded-concurrency `driveFetch` wrapper** (token bucket + jittered backoff all reads/writes funnel through) + a `LABS_DRIVE_AUTO_PULL_MIN_INTERVAL_MS >= 60s` regression floor. Kills the #28–32 API-abuse/ban class (no absolute rate governor today).                                                | Open — ban-risk floor                                                     |
| P2       | `test gap`          | **Guard-parity contract test** (`labsPortfolioDriveHookGuardrails.test.ts`): for every app, `downloadSidecars` tolerates a single 404 blob without aborting the batch or pausing sync (#12 — **live**, Zinebox/Lyrefly), and push has a filled>empty + post-upload size check (#14/#26).                                                   | Open — live: one trashed sidecar pauses a whole library's sync            |
| P2       | `missing invariant` | **Ancestry/provenance assertions pinned by argument** (#19/#20/#21): one shared `trashableRootIds` helper for delete+dedup, per-descendant provenance independent of OAuth scope, ancestor-set argument pinned in tests so a widening edit fails CI.
