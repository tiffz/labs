# Writing style

Applies to anything an agent writes: docs, commit messages, PR descriptions, retrospectives, code comments, chat responses — not just user-facing copy (see [`user-copy.md`](user-copy.md) for UI-specific conventions).

- Never use a long word where a short one will do.
- If it is possible to cut a word out, cut it out.
- Never use the passive where you can use the active.
